# 要件定義

## １.一言サービスコンセプト

- 原価高騰のピンチを、仕込み中の片手操作で『値上げかポーション調整か』の最適解に導く、飲食店専用の即決ナビゲーター

## 2.誰のどんな課題を解決するか

- 飲食店のオーナーや管理者でパソコンをじっくり開く時間はなく、キッチンの現場で仕込みをしながら、手元のスマートフォンでサクッと原価や利益を確認したいと考えている人を対象
- 食材費の高騰に対し、どのメニューを、いつ、いくら値上げするまたは仕様変更すべきかなどの意思決定が遅れ、利益を把握しにくくなっている。

## 3.なぜそれを解決したいのか

- 私の働いている店では、経営が苦しい時期もあり、オーナーが複数回変わっている。引き継ぎもうまくいかず、その度にやるべき作業が山のようにあった。食材の管理を任されていたこともあり、効率的で、簡単なシステムを作りたいと思った。

## 4.どうやって解決するのか

- 食材の価格、メニューの価格や目標減価率を登録。
- そのメニューが目標減価率を超えていないかを確認する。
- 超えていた場合には、メニューの値上げや食材のポーション量の調整を提案する。
 
## 5.機能要件

# 登録機能
- 食材の仕入れ価格、内容量を入力できる。
- 登録した食材をもとにメニュー作成して、販売価格と目標原価率を登録できる。

# 計算機能
- 食材の価格と量からその食材の１グラムあたりの原価を算出できる。
- メニューに登録した食材をもとに、そのメニューの原価を算出できる。

# アラート機能
- メニュー作成時や内容変更時に目標原価率を超えたメニューにアラートを出す。

# 編集機能
- 仕入れ価格、内容量やメニューの詳細などの変更をいつでもできる

# 提案機能
- 目標原価率を超えた超えたメニューに対して、メニューの販売価格と使用する食材のポーション量の変更の2種類のアプローチで、目標原価率を達成するように計算して、提案する。

## 6.非機能要件

- 使用者をパソコンをゆっくり操作する時間がないくらいの忙しい人やパソコンの操作が苦手な人を想定して、配色やボタン配置を見やすいものにする。
- いつでも手軽に使えるように、スマートフォンで操作することを前提にしたUIにする。
